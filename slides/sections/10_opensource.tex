\begin{frame}[t]{This Seminar's Code: A Reproducible Workflow}
  \begin{columns}[T]
    \begin{column}{0.52\textwidth}
      Everything you have seen is generated by the \texttt{gr} Python package in
      \textbf{one repository}:
      \begin{itemize}\small
        \item all four theories on \emph{one} parameter set and \emph{one}
        mechanical setting --- apples-to-apples;
        \item deliberately reduced-order (1D / thin-membrane, single stress
        component) so it runs in seconds;
        \item \texttt{docs/} lecture notes, \texttt{configs/} YAML input files
        driven by one \texttt{run.py} (edit a file or pass \texttt{--sweep} ---
        no Python to touch), \texttt{solutions/} that regenerate every figure.
      \end{itemize}
      \vspace{0.3em}
      {\color{AccentRed}\small \texttt{make figures} in \texttt{slides/}
      regenerates fig01--06 from the code, so the deck never drifts from the
      model.}
    \end{column}
